# RERA Easy — Maharashtra Real Estate Market Intelligence

Area-wise market analytics built from the MahaRERA project registry (54,281 projects, 2017–2026).
The project turns one raw Excel export into three branded deliverables for the RERA Easy
sales & business-development team.

## Deliverables (in the project root)

| File | What it is |
|------|------------|
| `RERA Market Analysis Dashboard.html` | Single-file **interactive dashboard**. Card-based drill-down (Division → District → Taluka → Village), an *Explore by* toggle (Geography / Project Type / Project Status / Bank), KPIs, animated donut charts, registrations trend, top-5 banks, a Land/Carpet/Built-Up area bar chart, and a **Generate Report** button (print / save as PDF). No internet required — everything is inline. |
| `RERA Easy - Maharashtra Market Analysis.docx` | Branded **Word report**: executive summary, supply mix, trend, division & district tables, top lending banks, delivery risk, and BD recommendations. |
| `RERA Easy - Developer Market Snapshot.html` | Branded **lead-gen mailer** for sales to send to developers: geographical market analytics on top, Active-vs-Lapsed insight, then the "RERA Easy can help" call-to-action. Print-ready. |

```
│   ├── Project RERA stats - Raw Data.xlsx  ← SOURCE data (54,281 rows, 21 cols)
│   ├── data.json                           ← cleaned, columnar (generated by prep.py)
│   └── stats.json                          ← aggregates (generated by build_stats.py)
└── src/
    ├── prep.py                 ← xlsx  → data/data.json   (clean + dictionary-encode)
    ├── build_stats.py          ← data.json → data/stats.json (aggregates for the report)
    ├── build_dashboard.py      ← template + data.json → dashboard HTML
    ├── dashboard_template.html ← dashboard source (has /*DATA_PLACEHOLDER*/ marker)

```

## Requirements

- **Python 3** with `openpyxl`  →  `pip install openpyxl`
- **Node.js** with the `docx` package  →  `npm install docx`  (creates `node_modules/`)

## Rebuild everything (run from the project root)

```bash
python src/prep.py            # Excel  -> data/data.json
python src/build_stats.py     # data.json -> data/stats.json
python src/build_dashboard.py # -> RERA Market Analysis Dashboard.html
node   src/build_doc.js       # -> RERA Easy - Maharashtra Market Analysis.docx
```

The developer mailer (`RERA Easy - Developer Market Snapshot.html`) is a static file —
edit `src/developer_snapshot.html` and copy it to the root, or just edit the root copy directly.

## Updating the data / using a new same-structure export

1. Drop in the new export. Either replace `data/Project RERA stats - Raw Data.xlsx`, **or** pass a
   path: `python src/prep.py "data/your new export.xlsx"`. The sheet may be named anything (the
   first sheet is used if there is no `Sheet1`). Keep the same 21 columns in the same order.
2. Re-run the four build commands above (`prep.py` → `build_stats.py` → `build_dashboard.py` → `build_doc.js`).
3. The dashboard and Word report **auto-derive** all counts and year references from the data —
   the project total, the year range (footer), the "launch backlog / year-to-date" notes, and the
   momentum window (last 3 full years vs prior 3) all recompute. No manual edits needed.

> The **developer mailer** (`src/developer_snapshot.html`) is hand-authored with baked-in numbers —
> it does **not** auto-refresh; update its figures manually if you send it with new data.

## Data notes

- Coverage is Maharashtra (plus a few Dadra & Nagar Haveli / Daman & Diu rows).
- **Cleaning:** carpet-area values above 10,000,000 and negative "avg days to complete" are
  treated as missing **for those metrics only** — the projects still count everywhere else.
- `2017` is inflated by the MahaRERA launch backlog; read trends from 2018 onward.
  `2026` is year-to-date / partial.
